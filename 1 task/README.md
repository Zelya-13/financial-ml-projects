# Классификация финансовых транзакций по категориям

**У компании есть выгрузка ежедневных транзакций:** дата/время, сумма, способ оплаты, категория и подкатегория покупки, текстовое примечание. В реальности разметку класса операции выполняют специалисты вручную — долго и с риском ошибок.

**Задача:** Каждая транзакция должна быть отнесена к одному из классов транзакции — например: «здоровье», «одежда», «налоги», «аренда», «командировочные расходы» и т.д. 

**Примечание:** в условии указан USA Banking Transactions Dataset, но фактически предоставлен Daily Household Transactions Dataset — работа выполнена по фактическим данным.

**Метрика качества:** Macro F1-score (среднее макро по всем категориям).

**Данные:**

- `Date`: дата и время совершения транзакции
- `Mode`: способ оплаты
- `Category`: каждая запись относится к определённой категории транзакций
- `Subcategory`: категории дополнительно разбиваются на подкатегории
- `Note`: краткая информация о проведённой операции
- `Amount`: сумма операции
- `Income/Expense`: каждая операция обозначается как расход или доход
- `Currency`: все операции фиксируются в официальной валюте Индии

**Особенность данных:** В данных присутствует сильный дисбаланс по целевому признаку (906 строк по самой частой категории, по 1 строке на самые нечастые категории).

**Что сделано:** заполнил пропуски в столбцах `Note` и `Subcategory`, изменил тип данных в `Date` с str на datetime, удалил дубликаты.
Затем подготовил данные для обучения моделей:

- Tfidf на `Note`
- OHE на категориальных признаках
- StandardScaler на `Amount`
- извлёк новые признаки из `Date` (день недели, месяц, выходной)
- сгруппировал редкие классы (меньше 10 строк) в категорию Other

После чего сравнил три модели: логистическую регрессию, случайный лес и CatBoost.

**Результат:** лучшей моделью стала логистическая регрессия — macro F1 = 0.495 на тестовой выборке. Метрика упирается в потолок по двум причинам. Первая — ограничение данных: на кросс-валидации все три модели показали около 0.5, то есть результат задаёт не выбор модели, а информативность признаков. Вторая — редкие классы: на категориях с единичными примерами модель предсказывает плохо, а macro F1 штрафует за это сильнее всего.

**Стек** — Python, pandas, matplotlib, scikit-learn, CatBoost.

**Как запустить:** Открыть Task_1.ipynb, положить в ту же папку Daily Household Transactions.csv, нажать Run All.
