# Анализ банковских транзакций

**Описание задачи:** У компании есть выгрузка транзакций (аналог бухгалтерских проводок): дата, сумма, тип операции (дебет/кредит или Debit/Credit), контрагент (merchant), категория расходов (например, еда, транспорт, подписки и т.п.), возможно описание операции и способ оплаты.

**Задача:** Каждая транзакция должна быть отнесена к одной из категорий транзакций — например: Food, Transport, Entertainment, Grocery, Electronics и т.д.

**Примечание:** в условии указан Daily Household Transactions Dataset, но фактически предоставлен USA Banking Transactions Dataset — работа выполнена по фактическим данным, также как в 1 задании.

**Метрика качества:** Macro F1-score (среднее макро по всем категориям).

**Данные:**

A. Сведения о транзакции:

- `Transaction_ID`: уникальный идентификатор каждой транзакции
- `Transaction_Date`: дата и время транзакции
- `Transaction_Amount`: денежная сумма транзакции
- `Transaction_Type`: тип транзакции (Debit или Credit)

B. Информация о клиенте:

- `Customer_Age`: возраст клиента (18–70)
- `Customer_Gender`: пол клиента (Male, Female, Others)
- `Customer_Income`: годовой доход клиента
- `Account_Balance`: остаток на счете после транзакции

C. Классификация:

- `Category`: классификация транзакций по соответствующим категориям, таким как Food, Transport, Entertainment, Grocery, Electronics и т.д.

D. Информация о продавце и способе оплаты:

- `Merchant_Name`: название продавца или поставщика
- `Payment_Method`: способ оплаты (Credit Card, Debit Card, Cash, Online Transfer и т.д.)

E. Дополнительные сведения:

- `City`: место проведения транзакции (крупные города США)
- `Fraud_Flag`: указывает, была ли транзакция помечена как мошенническая
- `Transaction_Status`: статус транзакции (Success, Failed, Pending)
- `Loyalty_Points_Earned`: бонусные баллы, начисленные за транзакцию
- `Discount_Applied`: логическое значение, указывающее, была ли применена скидка

**Особенность данных:** Данные по ключевому признаку распределены более менее равномерно (в диапазоне от 347 до 417), по всем числовым столбцам и большинству категориальных столбцов данные распределены также равномерно. Аномалий, дубликатов и пропусков в данных нет.

**Что сделано:** изменил тип данных в Date с str на datetime. Затем подготовил данные для обучения моделей:

- OHE на категориальных признаках
- StandardScaler на числовых признаках
- извлёк новые признаки из Date (день недели, месяц, выходной)
- убрал признаки `Transaction_Description`, `Merchant_Name` и `Customer_Occupation`, исходя из результатов baseline теста на каждом признаке по отдельности

После чего сравнил три модели: логистическую регрессию, случайный лес и LightGBM.

**Результат:** лучшей моделью стал случайный лес — macro F1 = 0.0748 на тестовой выборке. Метрика настолько низкая из-за синтетических данных. Синтетичность данных подтверждают:

1. Бессмысленные комментарии к транзакциям в столбце `Transaction_Description`
2. Отсутсвие разницы средних значений всех числовых признаков, сгруппированных по целевому признаку
3. Равномерность распределения всех признаков

**Стек** — Python, pandas, matplotlib, scikit-learn, LightGBM.
